### Purpose

The **Student Dashboard** is the primary interface students see after logging in. It provides an overview of enrolled courses, progress, upcoming deadlines, and quick access to learning resources. This module also includes lesson navigation within courses.

### Business Value

- **User Experience:** First impression matters - dashboard sets the tone
- **Engagement:** Easy navigation = higher course completion rates
- **Retention:** Clear overview of progress motivates students
- **Efficiency:** Quick access to learning resources saves time

### Success Metrics

- Dashboard load time: <1.5 seconds
- Navigation efficiency: Find next lesson in <3 clicks
- User satisfaction with dashboard: >4.5/5
- Daily active users (return to dashboard): >60%

### Story 3.1.2: My Courses List (Enrolled Courses with Progress)

**Story Points:** 5  
**Priority:** ðŸ”´ Critical  
**Sprint:** Sprint 3

#### User Story

```
AS A student
I WANT TO see all my enrolled courses in one place
SO THAT I can easily switch between courses
```

#### Acceptance Criteria

**AC1: My Courses Page**

- GIVEN I click "My Courses" in navigation
- WHEN I navigate to `/student/courses`
- THEN I see:
  - Page title: "My Courses"
  - List of enrolled courses (grid or list view)
  - Each course card shows:
    - Cover image
    - Course title
    - Progress bar + percentage
    - Last accessed date (e.g., "Last accessed 2 days ago")
    - "Continue Learning" button

**AC2: Course Filtering/Sorting**

- Filter by progress:
  - All courses
  - In Progress (1%-99%)
  - Not Started (0%)
  - Completed (100%)
- Sort by:
  - Recently accessed
  - Progress (high to low)
  - Title (A-Z)

**AC3: Continue Learning Button**

- GIVEN I click "Continue Learning"
- WHEN the button is clicked
- THEN:
  - Redirect to last accessed lesson (if exists)
  - OR redirect to next incomplete lesson
  - OR redirect to first lesson (if none accessed yet)

### Story 3.1.4: Lesson Navigation (Prev/Next, Sidebar)

**Story Points:** 5  
**Priority:** ðŸ”´ Critical  
**Sprint:** Sprint 3

#### User Story

```
AS A student
I WANT TO easily navigate between lessons
SO THAT I can move through the course smoothly
```

#### Acceptance Criteria

**AC1: Lesson Page Layout**

- GIVEN I am viewing a lesson (`/courses/:courseId/learn/:lessonId`)
- WHEN the page loads
- THEN I see:
  - **Left Sidebar (Course Navigation):**
    - Course title (at top)
    - Progress bar (% complete)
    - Collapsible sections
    - Lesson list (same as course detail view)
    - Current lesson highlighted
  - **Main Content Area:**
    - Lesson content (video/quiz/reading/assignment based on type)
  - **Bottom Navigation:**
    - "â† Previous Lesson" button (disabled if first lesson)
    - "Next Lesson â†’" button (disabled if last lesson or next is locked)
    - "Mark as Complete" button (if not complete)

**AC2: Sidebar Navigation**

- GIVEN I click a lesson in sidebar
- WHEN the lesson is unlocked
- THEN:
  - Navigate to that lesson
  - Sidebar remains visible
  - New lesson loads in main content area

**AC3: Previous/Next Buttons**

- GIVEN I click "Next Lesson"
- WHEN next lesson is unlocked
- THEN:
  - Navigate to next lesson
  - URL updates
  - Content updates
- GIVEN next lesson is locked
- WHEN I try to click "Next Lesson"
- THEN:
  - Button disabled
  - Tooltip: "Complete current lesson and wait for instructor to unlock"

**AC4: Mobile View**

- Sidebar collapsible on mobile
- Hamburger menu icon to toggle sidebar
- Bottom navigation always visible

### Story 3.1.5: Learning Interface (Video/Quiz/Material Renderer)

**Story Points:** 8  
**Priority:** ðŸ”´ Critical  
**Sprint:** Sprint 3

#### User Story

```
AS A student
I WANT TO view lesson content in a consistent interface
SO THAT I have a seamless learning experience across content types
```

#### Acceptance Criteria

**AC1: Content Renderer (Based on Lesson Type)**

- System detects lesson type and renders appropriate component:
  - **Video:** YouTube embed (from Module 4, Story 2.2.1)
  - **Quiz:** Quiz interface (from Module 4, Stories 2.2.4-2.2.9)

  - Content area (center)

**AC3: Progress Tracking Integration**

- As student interacts with content:
  - Video: Track watch time
  - Quiz: Track answers
  - Reading: Track scroll depth
  - Assignment: Track submission status
- Progress saved automatically (from Module 7)
