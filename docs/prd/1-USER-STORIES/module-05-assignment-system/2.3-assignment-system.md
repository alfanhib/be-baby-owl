## ðŸ“‹ Module Overview

### Purpose

This module enables **instructors to create assignments, students to submit work, and instructors to grade submissions**. Assignments are a core component of instructor-supervised learning, providing opportunities for students to apply knowledge and receive personalized feedback.

### Business Value

- **Learning Outcomes:** Assignments reinforce learning through practice
- **Instructor Value:** Grading assignments is a key instructor activity
- **Student Engagement:** Personalized feedback increases student satisfaction
- **Assessment:** Measure student progress beyond quizzes

### Success Metrics

- Assignment submission rate: >80% (students submit on time)
- Average grading turnaround time: <48 hours
- Instructor grading efficiency: Grade 1 assignment in <5 minutes
- Student satisfaction with feedback: >4.5/5

### Story 2.3.1: Create Assignment (Instructor, with Rubric)

**Story Points:** 8  
**Priority:** ðŸŸ  High  
**Sprint:** Sprint 6

#### User Story

```
AS AN instructor
I WANT TO create assignments for students
SO THAT students can practice skills and I can assess their understanding
```

#### Acceptance Criteria

**AC1: Access Assignment Creation**

- GIVEN I am editing a course (as instructor/admin)
- WHEN I add a new lesson and select type: "Assignment"
- THEN I am redirected to assignment creation form

**AC2: Assignment Creation Form**

- Form includes:
  - **Assignment Title** (text input, required, max 200 chars)
  - **Instructions** (rich text editor, required)
    - Support: headings, bold, italic, lists, images, code blocks
    - Example: "Build a React Native app that displays a list of users..."
  - **Submission Type** (dropdown):
    - File upload (PDF, images, ZIP files)
    - Text submission (essay, code, etc.)
    - Link submission (GitHub repo, Google Drive, etc.)
    - Multiple types allowed (checkboxes)
  - **Max File Size** (if file upload, default: 10MB, max: 50MB)
  - **Allowed File Types** (if file upload, e.g., PDF, DOCX, ZIP, JPG, PNG)
  - **Due Date** (date picker, optional)
  - **Points** (number input, e.g., 100 points, default: 100)
  - **Rubric** (optional, for grading criteria):
    - Criteria name (e.g., "Code Quality")
    - Points (e.g., 30 points)
    - Description (e.g., "Code is clean, well-organized, follows best practices")
    - Add multiple criteria
  - **Auto-Unlock Next Lesson** (checkbox, default: false)
    - If checked, next lesson unlocks after assignment is graded
  - "Save Assignment" button

**AC3: Create Assignment (Save)**

- GIVEN I fill all required fields
- WHEN I click "Save Assignment"
- THEN:
  - API call to `POST /api/lessons/:lessonId` (with assignment data)
  - Assignment created
  - Success toast: "Assignment created successfully!"
  - Assignment appears in course structure

**AC4: Rubric Builder**

- GIVEN I want to add grading criteria
- WHEN I click "Add Criteria" in rubric section
- THEN:
  - New criteria row added
  - Fields: Criteria name, Points, Description
  - Can add up to 10 criteria
  - Total points auto-calculated (sum of all criteria)
  - Total should match "Points" field (show warning if mismatch)

**AC5: Default Rubric**

- If instructor doesn't create custom rubric:
  - Use simple rubric: Single criteria "Assignment Quality" (100 points)

---

#### Edge Cases

**E1: Due Date in Past**

- If instructor sets due date in past:
  - Show warning: "Due date is in the past. Students will see this as overdue."
  - Allow (don't block, instructor may be creating assignment retroactively)

**E2: File Size Too Large**

- If instructor sets max file size >50MB:
  - Show error: "Max file size cannot exceed 50MB"

**E3: Rubric Points Mismatch**

- If rubric criteria sum â‰  assignment points:
  - Show warning: "Rubric criteria total (85 pts) doesn't match assignment points (100 pts)"
  - Allow (don't block, instructor may adjust)

**E4: No Due Date**

- If no due date set:
  - Assignment never considered "overdue"
  - Students can submit anytime

---

#### Acceptance Criteria

**AC1: Submission Form**

- GIVEN I am viewing an assignment
- WHEN I scroll to submission section
- THEN I see submission form based on allowed types:
  - **If "File Upload" allowed:**
    - File upload area (drag & drop or click to browse)
    - Max file size shown (e.g., "Max 10MB")
    - Allowed file types shown (e.g., "PDF, ZIP, JPG, PNG")
  - **If "Text Submission" allowed:**
    - Textarea (rich text editor)
    - Character counter (max 10,000 chars)
  - **If "Link Submission" allowed:**
    - Text input for URL
    - Placeholder: "https://github.com/username/repo"
  - Optional comment field (text area, "Add a comment for your instructor")
  - "Submit" button

**AC2: File Upload**

- GIVEN I select "file upload" submission
- WHEN I drag & drop or select a file
- THEN:
  - File validated:
    - Size â‰¤ max file size â†’ proceed
    - Size > max â†’ error: "File too large. Max 10MB."
    - Type not allowed â†’ error: "File type not allowed. Allowed: PDF, ZIP, JPG, PNG"
  - File preview shown (thumbnail for images, file icon for others)
  - File name displayed
  - "Remove" button to delete and upload again

**AC3: Submit Assignment**

- GIVEN I have filled submission form (file/text/link)
- WHEN I click "Submit"
- THEN:
  - Confirmation modal: "Submit assignment? You can resubmit until the due date."
  - "Cancel" and "Confirm Submit" buttons
- WHEN I confirm
- THEN:
  - API call to `POST /api/assignments/:assignmentId/submissions`
  - File uploaded to S3/Cloudinary (if file submission)
  - Submission saved to database
  - Success toast: "Assignment submitted successfully!"
  - Submission status changes to "Submitted on [date]"
  - "Submit" button changes to "View Submission" or "Resubmit"
  - Email notification sent to instructor: "New assignment submission from [Student]"

**AC4: Resubmit Assignment (Before Due Date)**

- GIVEN I have already submitted
- WHEN I click "Resubmit" (before due date)
- THEN:
  - Show previous submission (file/text/link)
  - Allow editing/replacing
  - On submit, new version saved (old version archived)
  - Submission count incremented (e.g., "Submission #2")

**AC5: Resubmit After Due Date**

- GIVEN due date has passed
- WHEN I try to resubmit
- THEN:
  - Show warning: "Due date has passed. Resubmission may be considered late."
  - Allow resubmit (unless instructor disabled)
  - Mark as "Late Submission"

**AC6: View Own Submission**

- GIVEN I have submitted
- WHEN I click "View Submission"
- THEN I see:
  - Submitted file/text/link (read-only)
  - Submission date and time
  - Submission comment
  - Status:
    - "Pending Grading" (if not graded yet)
    - "Graded: 85/100" (if graded)
  - Instructor feedback (if graded)
  - Grade breakdown (if rubric used)

---

#### Edge Cases

**E1: Submit Without Filling Form**

- If student clicks "Submit" with empty form:
  - Show error: "Please upload a file, enter text, or provide a link"

**E2: Network Error During Upload**

- If upload fails:
  - Show error: "Upload failed. Please try again."
  - Don't save submission (atomic operation)

**E3: File Upload Interrupted**

- If upload interrupted (lost connection):
  - Show error: "Upload interrupted. Please try again."
  - Implement retry logic (background)

**E4: Multiple Files**

- For MVP: Single file upload only
- Future: Allow multiple files (ZIP archive recommended for now)

**E5: Submit After Grading**

- If instructor already graded:
  - Show warning: "This assignment has been graded. Resubmission will require re-grading."
  - Allow resubmit (grade reset to "Pending")

### Story 2.3.5: Instructor Grading Interface (View Submission, Score)

**Story Points:** 8  
**Priority:** ðŸŸ  High  
**Sprint:** Sprint 7

#### User Story

```
AS AN instructor
I WANT TO grade student assignment submissions
SO THAT students receive feedback and scores
```

#### Acceptance Criteria

**AC1: Pending Submissions Queue**

- GIVEN I am logged in as instructor
- WHEN I navigate to "Assignments to Grade" (in instructor dashboard)
- THEN I see:
  - List of pending submissions (not graded yet)
  - Each entry shows:
    - Student name + avatar
    - Assignment title
    - Submitted date (e.g., "2 days ago")
    - "Grade" button
  - Filter by:
    - Class (dropdown)
    - Assignment (dropdown)
    - Submission date (newest first, oldest first)

**AC2: Grading Interface**

- GIVEN I click "Grade" on a submission
- WHEN the grading page loads
- THEN I see:
  - Student name
  - Assignment title + instructions (collapsible)
  - Submission content:
    - If file: Download link + file preview (PDF inline, image inline, others download only)
    - If text: Full text displayed
    - If link: Clickable link (opens in new tab)
  - Student comment (if provided)
  - Grading section (right sidebar or below):
    - **Score Input:** Number input (e.g., "85" out of 100)
    - **Rubric Breakdown** (if rubric exists):
      - Each criteria with score input
      - Example: "Code Quality: \_\_\_\_ / 30"
      - Auto-calculate total
    - **Feedback** (textarea, rich text):
      - Provide detailed feedback to student
      - Example: "Great job on the UI! However, the state management could be improved..."
    - "Save as Draft" button (save grade without notifying student)
    - "Submit Grade" button (finalize grade, notify student)

**AC3: Rubric Grading**

- GIVEN assignment has rubric
- WHEN I grade submission
- THEN:
  - Rubric criteria shown
  - Enter score for each criteria (e.g., Code Quality: 25/30)
  - Total auto-calculated (sum of all criteria)
  - If total â‰  assignment points, show warning

**AC4: Submit Grade**

- GIVEN I have entered score and feedback
- WHEN I click "Submit Grade"
- THEN:
  - Confirmation: "Submit grade? Student will be notified."
  - If confirmed:
    - Grade saved to database
    - Assignment marked as "Graded"
    - Email sent to student: "Your assignment '[Title]' has been graded"
    - XP awarded to student (+50 XP for graded assignment, regardless of score)
    - Success toast: "Grade submitted successfully!"
    - Redirected to next pending submission (if any) or back to queue

**AC5: Save as Draft**

- GIVEN I want to save grade without finalizing
- WHEN I click "Save as Draft"
- THEN:
  - Grade saved (not visible to student yet)
  - Can return later to finish grading
  - No email sent to student

---

#### Edge Cases

**E1: Student Resubmits After Grading**

- If student resubmits after grade given:
  - Grade reset to "Pending"
  - Old grade archived
  - Instructor notified: "Student resubmitted assignment"

**E2: Rubric Score Exceeds Max**

- If instructor enters 35/30 for a criteria:
  - Show warning: "Score cannot exceed maximum (30)"
  - Block submission until corrected

**E3: Grade Without Feedback**

- Allow grading without feedback (optional)
- Show warning: "No feedback provided. Students appreciate feedback!"
  - Allow proceeding
